# Hound.Trading — Trading Pack

The first pack: autonomous trading hounds using Alpaca Markets paper trading.

## Hounds
- `AnalysisHound` — Analyses market data, produces recommendations with confidence scores
- `StrategyHound` — Determines trading strategy (buy/sell/hold) based on analysis
- `RiskHound` — Evaluates trades against risk limits, approves/rejects
- `ExecutionHound` — Executes approved trades via Alpaca API

## Workflow
`AnalysisHound → StrategyHound → RiskHound → ExecutionHound`
Defined as an AF graph-based workflow in `Workflows/TradingWorkflow.cs`.
Optional human-in-the-loop checkpoint before execution.

## Key Patterns
- Each hound is an AF Agent (using `Microsoft.Agents.AI`)
- Hound configs are externalized in `Config/{HoundName}.json`
- All hounds inject `IActivityLogger` for activity logging
- `AlpacaService` wraps the `Alpaca.Markets` NuGet (thin adapter)
- `Program.cs` is a .NET Generic Host running the workflow on a schedule

## Dependencies
- `Hound.Core` — Models, logging, LLM client
- `Alpaca.Markets` — Paper trading API
- `Microsoft.Agents.AI` (transitive via Hound.Core)

## Do NOT
- Call Alpaca directly from hounds — use `IAlpacaService`
- Hardcode trading parameters — use config files
- Run live trades — paper trading only
