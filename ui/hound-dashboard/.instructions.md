# hound-dashboard — Angular 19+ SPA

Monitoring dashboard for Hound AI packs and hound activity.

## Pages
- **Dashboard** (`/`) — Overview cards for each pack
- **Pack Detail** (`/packs/:id`) — Hound list + real-time activity feed
- **Activity Log** (`/activity`) — Filterable, paginated activity table
- **Tuner** (`/tuner`) — TunerHound experiment review (Wave 4)

## Services
- `ApiService` — HTTP client wrapping REST endpoints at `http://localhost:5000`
- `SignalrService` — SignalR connection to `/hubs/activity` for real-time updates

## Patterns
- Standalone components (no NgModules)
- Signals where appropriate for reactive state
- OnPush change detection preferred
- SCSS for styling (minimal, no heavy UI framework)
- Models in `src/app/models/` match C# domain models

## Key Dependencies
- `@microsoft/signalr` — SignalR client
- `@angular/common/http` — HTTP client
- `@angular/router` — Client-side routing

## API Contract
- `GET /api/packs` → `Pack[]`
- `GET /api/packs/{id}` → `Pack`
- `GET /api/packs/{packId}/hounds` → `HoundInfo[]`
- `GET /api/activity?pack=&hound=&from=&to=&page=&pageSize=` → `PagedResult<ActivityLog>`
- SignalR Hub at `/hubs/activity` — methods: `SubscribeToPack`, `UnsubscribeFromPack`, events: `OnActivity`

## Do NOT
- Import `BrowserModule` (use standalone components)
- Add server-side rendering (SSR is disabled)
- Hardcode API URLs (use environment config)
